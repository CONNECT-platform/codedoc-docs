linkButtons

You can add buttons to your documents like this:

some-doc.md
1> :Buttons

2> > :Button label=Click Me!, url=https://www.google.com

3>

4> > :Button label=Click Me Too!, url=https://www.github.com

Which looks like this:

Click Me!Click Me Too!

You can create icon buttons as well. By default material icons are used:

some-doc.md
1> :Buttons

2> > :Button icon=true, label=android, url=https://www.google.com

3>

4> > :Button icon=true, label=code, url=https://www.github.com

5>

6> > :Button label=GitHub, url=https://www.github.com

Which looks like this:

androidcodeGitHub

You can add a copy button after a code element. This button would copy the contents of the code element.

some-doc.md
1> :Buttons

2> > :CopyButton

linkTabs

You can add tabbed content like this:

some-doc.md
1> :Tabs

2> > :Tab title=First Tab

3> >

4> > So this is the content of the first tab. Lets even have some code here:

5> > ```tsx | index.tsx

6> > import { Renderer } from '@connectv/html';

7> >

8> > const renderer = new Renderer();

9> > renderer.render(<div>Hellow World!</div>).on(document.body);

10> > ```

11>

12> > :Tab title=Second Tab

13> >

14> > Perhaps some other content here, maybe some more code?

15> >

16> > ```tsx | another.tsx

17> > import { Renderer } from '@connectv/html';

18> > import { timer } from 'rxjs';

19> >

20> > const renderer = new Renderer();

21> > renderer.render(<div>You have been here for {timer(0, 1000)} second(s).</div>)

22> > .on(document.body);

23> > ```

linkCollapse

You can add collapsible sections like this:

some-doc.md
1> :Collapse label=Collapsible content (click to open)

2>

3> This content is collapsed by default. You can write _any_ markdown syntax you would

4> like here as this is simply just an enhanced `block quote` element. You can even have lists:

5> - with multiple

6> - items and stuff

7>

8> > :Collapse label=Or nested collapsible content (click to open)

9> >

10> > > :Collapse label=Collapception

11> > >

12> > >  To any depth that your heart might desire.

13>

14> This component is particularly useful in the table of contents (the left-side menu, activatable by

15> clicking on the hamburger menu in the footer), when you have got many documents and you would want to

16> neatly categorize them.

linkDark/Light Content

If you have some content that differs in light mode vs in dark mode, you can use DarkLight component:

some-doc.md
1> :DarkLight

2> > :InDark

3> >

4> > We are SO DARK! This content is only shown in dark-mode. Switch to light-mode

5> > by clicking on the dark/light toggle in the footer to see the light-mode specific content.

6>

7> > :InLight

8> >

9> > LIGHT bless you! This content is only shown in light-mode. Switch to dark-mode

10> > by clicking on the dark/light toggle in the footer to see the dark-mode specific content


We are SO DARK! This content is only shown in dark-mode. Switch to light-mode by clicking on the dark/light toggle in the footer to see the light-mode specific content.

LIGHT bless you! This content is only shown in light-mode. Switch to dark-mode by clicking on the dark/light toggle in the footer to see the dark-mode specific content

If you are wondering why someone would EVER need such a thing, well I did. Because I wanted to display different banner images based on dark-mode / light-mode:

1> :DarkLight

2> > :InLight

3> >

4> > ![header](https://raw.githubusercontent.com/CONNECT-platform/codedoc/master/repo-banner.svg?sanitize=true)

5>

6> > :InDark

7> >

8> > ![header](https://raw.githubusercontent.com/CONNECT-platform/codedoc/master/repo-banner-dark.svg?sanitize=true)

Yeah I know the life of an idealist is never easy.
